You are acting as a senior frontend engineer and UI systems architect.

I am building a professional fintech website with multiple pages (landing page, dashboard, portfolio recommendation pages, analytics sections, forms, charts, pricing sections, footer, navigation, etc).

I want to completely refactor the typography system of the entire website so that ALL text elements across every page are controlled centrally through CSS classes and can be globally changed later by only editing font imports and font-family declarations.

Your task is NOT to redesign the website.

Your task is ONLY to restructure typography architecture.

Follow these instructions extremely carefully.

---

GOAL:

Create a semantic typography system where every text element in the website is classified by its FUNCTION and CONTEXT, not by arbitrary styling.

I should later be able to change the entire website font system by modifying ONLY ONE CSS file.

---

STEP 1 — ANALYZE ENTIRE CODEBASE

Scan ALL pages/components.

Understand the CONTEXT of each text element before assigning classes.

Do NOT blindly assign classes based on HTML tags.

Example:

* An h1 inside hero section is NOT the same as an h1 inside a card.
* Small text in footer is NOT the same as small text under a graph.
* A number showing investment return is NOT normal body text.
* Graph axis labels are NOT paragraph text.
* Dashboard metric percentages are NOT regular numbers.

You must understand the purpose of the text.

---

STEP 2 — CREATE UNIVERSAL TYPOGRAPHY SYSTEM

Create reusable semantic classes.

Use ONLY these categories.

---

.text-section-subtitle
Section subheading text

.text-paragraph
Normal descriptive content

.text-highlight
Highlighted important words

---

STEP 3 — MODIFY HTML/JSX/COMPONENTS

Go through ALL pages.

Replace old inconsistent typography classes.

Assign semantic typography classes ONLY after understanding actual context.

Do NOT assign classes based on tag names.

Example:

Wrong:

All h2 → text-section-title

Correct:

h2 inside card → text-card-title

h2 inside analytics section → text-section-title

---

STEP 4 — CENTRALIZE CSS

Create ONE typography stylesheet.

Example:

typography.css

This file should contain ALL font-family declarations.

Example structure:

.text-brand {
font-family: var(--font-brand);
}

.text-hero-title {
font-family: var(--font-heading);
}

.text-paragraph {

}

}

.text-footer-text {

}

At the top of CSS file create variables ONLY.

Example:

All classes must use ONLY these variables.

Never hardcode fonts directly inside classes.
